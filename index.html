Core AWS Services

Key services you'll use every day in production
EC2
Elastic Compute Cloud
Scalable virtual machines. Choose instance types optimized for compute, memory, GPU, or storage.
S3
Simple Storage Service
Object storage with 99.999999999% durability. Store and retrieve any amount of data, anytime.
VPC
Virtual Private Cloud
Isolated network environment. Define subnets, route tables, gateways, and security groups.
RDS
Relational Database Service
Managed MySQL, PostgreSQL, MariaDB, Oracle, SQL Server, Aurora with automated backups.
IAM
Identity & Access Management
Manage users, groups, roles, and fine-grained permissions across all AWS resources.
Lambda
Serverless Functions
Run code without provisioning servers. Pay per invocation. Supports 12+ runtimes.
EKS
Elastic Kubernetes Service
Managed Kubernetes. Deploy, scale, and manage containerized workloads at scale.
CloudFront
Content Delivery Network
400+ edge locations globally for low-latency content delivery and DDoS protection.
SQS
Simple Queue Service
Fully managed message queuing for microservices. Decouple and scale distributed systems.
ECS
Elastic Container Service
Scalable Docker container orchestration on managed clusters with Fargate or EC2 launch types.
DynamoDB
NoSQL Database
Single-digit millisecond latency at any scale. Serverless key-value and document database.
Route 53
DNS & Traffic Routing
Highly available DNS service with health checks, failover, and geolocation routing.

Core Concepts

Fundamentals every AWS engineer must internalize
01
Regions & Availability Zones
AWS is split into geographic Regions (us-east-1, ap-south-1…). Each Region has multiple isolated AZs. Deploy across AZs for true high availability.
Infrastructure
02
IAM Least Privilege
Always grant the minimum permissions required. Prefer roles over long-term access keys. Enable MFA on root and all admin accounts.
Security
03
Well-Architected Framework
Six pillars: Operational Excellence, Security, Reliability, Performance, Cost Optimization, Sustainability. The blueprint for cloud-native systems.
Architecture
04
Auto Scaling & Load Balancing
Auto Scaling Groups dynamically adjust EC2 capacity. Pair with ALB or NLB to distribute traffic and automatically replace unhealthy instances.
Reliability
05
ARN — Amazon Resource Name
Every AWS resource has a unique ARN. Format: arn:partition:service:region:account-id:resource. Required in IAM policies and CLI targeting.
Core
06
VPC & Subnet Design
A VPC is your private network in AWS. Public subnets connect via Internet Gateway. Private subnets use NAT Gateways for outbound-only access.
Networking
07
Pay-as-you-go Pricing
Pay only for compute time, storage GBs, and API calls consumed. Use Reserved Instances or Savings Plans for up to 72% discount on predictable workloads.
Cost
08
Infrastructure as Code
CloudFormation and CDK let you define AWS resources as versioned code. Provision stacks repeatably, roll back on failure, and enforce consistency.
DevOps
09
Event-Driven Architecture
Wire EventBridge, SNS, SQS, and Lambda together for decoupled systems. S3 events trigger Lambda; SQS buffers spiky traffic; SNS fans out to multiple consumers.
Architecture
Pro Tip
Always configure the CLI first with aws configure. Use named profiles (--profile dev, --profile prod) to switch between AWS accounts. Credentials live in ~/.aws/credentials. Use --output table for human-readable results in the terminal.
